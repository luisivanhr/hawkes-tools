# Hawkes-Tools API Support

Audit date: 2026-07-05.

The current package is a standalone pure-Python/NumPy/SciPy/Numba library. It
keeps a broad production API for Hawkes processes, GLM helpers, solvers,
proximal operators, preprocessing, survival analysis, robust models, metrics,
plotting, random helpers, array serialization, and vendored datasets.

The source-backed equivalence ledger reads a frozen manifest of 171 Hawkes
behavior cases without requiring any external compiled Hawkes package. Current
classification is 171 `pass`, 0 `xfail_equivalence_gap`, and 0
`skip_optional_backend`.

| Area | Supported hawkes-tools surface | Notes / exclusions |
| --- | --- | --- |
| Simulations | `SimuPoissonProcess`, `SimuInhomogeneousPoisson`, `SimuHawkes`, `SimuHawkesExpKernels`, `SimuHawkesSumExpKernels`, and `SimuHawkesMulti`. | Common point-process utilities are present: `track_intensity`, `is_intensity_tracked`, `store_compensator_values`, `set_timestamps`, `reset`, `threshold_negative_intensity`, and `SimuHawkes.check_parameters_coherence`. Source-backed stochastic simulation tests preserve statistical and stopping assertions; exact compiled RNG stream parity is not claimed. Fresh constant-baseline `SimuHawkes` runs with zero/exponential/time-function kernels use exact recursive exponential updates plus JIT-backed finite-support time-function windows, avoiding old full-history scans. Public deep import paths such as `hawkes_tools.hawkes.simulation.base` resolve through compatibility aliases. |
| Kernels | `HawkesKernel0`, `HawkesKernelExp`, `HawkesKernelSumExp`, `HawkesKernelPowerLaw`, `HawkesKernelTimeFunc`, and the convenience `HawkesKernel` export. | Value, primitive, norm, support, string, and TeX-style string behavior is covered by compatibility tests. Raw attributes replace some read-only property wrappers from compiled implementations. |
| Models | `ModelHawkesExpKernLogLik`, `ModelHawkesExpKernLeastSq`, `ModelHawkesSumExpKernLogLik`, and `ModelHawkesSumExpKernLeastSq`. | `fit`, `incremental_fit`, `loss`, `grad`, `loss_and_grad`, `hessian`, `hessian_norm`, and `cast_period_length` are available. Deterministic model formulas are the strongest equivalence area. |
| Learners | `HawkesExpKern`, `HawkesSumExpKern`, `HawkesEM`, `HawkesADM4`, `HawkesBasisKernels`, `HawkesConditionalLaw`, `HawkesCumulantMatching`, `HawkesCumulantMatchingTf`, `HawkesCumulantMatchingPyT`, and `HawkesSumGaussians`. | Constructors mostly preserve the established parameter names and defaults. Fit/score/kernel accessors, learner plotting helpers, objective methods, cumulant computation, and PyTorch-backed cumulant solve variants are covered by source-backed tests. `HawkesCumulantMatchingTf` is kept only as a legacy class name that delegates to the PyTorch backend; TensorFlow is not a supported production backend. |
| GLM models, learners, and generic simulation helpers | `hawkes_tools.linear_model` exports linear, logistic, Poisson, hinge, quadratic-hinge, and smoothed-hinge models, plus linear/logistic/Poisson learners and simulators. `hawkes_tools.simulation` exports `features_normal_cov_uniform`, `features_normal_cov_toeplitz`, `weights_sparse_exp`, and `weights_sparse_gauss`. | Dense loss/gradient loops are Numba JIT-backed, sparse logistic paths use SciPy CSR algebra, coefficient vectors use the established weights-first/intercept-last convention, and learner/model methods validate fit state, shapes, finite inputs, solver settings, and penalties `none`, `l1`, `l2`, `elasticnet`, `tv`, and `binarsity`. `binarsity` requires explicit `blocks_start` and `blocks_length`. |
| Prox and solvers | `hawkes_tools.prox` and `hawkes_tools.solver` expose pure-Python `Prox*` operators plus `GD`, `AGD`, `BFGS`, `GFB`, `SCPG`, `SGD`, `AdaGrad`, `SVRG`, `SAGA`, and `SDCA` style `set_model(...).set_prox(...).solve(...)` flows. | Solver settings, stochastic option strings, starts, malformed models, and malformed prox inputs fail with explicit validation errors. SVRG and SAGA use sparse stochastic mini-batches and variance-reduced update formulas; compiled asynchronous worker semantics are intentionally out of scope. |
| Plotting | `hawkes_tools.plot` and `hawkes_tools.hawkes.plot` export Hawkes kernels/norms/baselines, basis kernels, point processes, QQ diagnostics, time functions, optimization history, estimated intensity, and generic `stems`. | Optional Bokeh rendering for `stems` is kept behind Bokeh imports; Matplotlib behavior is covered by tests. Full visual pixel parity is not claimed. |
| Datasets | `hawkes_tools.datasets` loads the bundled Hawkes Bund, Adult, Covtype, IJCNN, KDD2010, Reuters, and Abalone payloads. URL reputation data is handled as a managed external dataset. | Metadata documents source, format, shape, nonzero counts, byte size, and SHA-256 for each vendored payload. Tests verify every vendored manifest entry is present and matches its recorded size and checksum. |
| Utility methods | `TimeFunction` is exported from `hawkes_tools` and `hawkes_tools.hawkes`; `BaseEstimator.get_params` and `set_params` are available. | `TimeFunction` covers constants, interpolation modes, border modes, `value`, `dt`, `get_norm`, `primitive`, and `future_bound`. Compiled memory/layout details and exact read-only attribute semantics are intentionally not mirrored. |

Mandatory NumPy/Numba acceleration is part of the standalone runtime contract:
the package requires Numba and dispatches hot loops through JIT helpers.
Python/NumPy reference formulas remain in-tree for equivalence assertions.
Benchmark smoke coverage reports cold, warm, and direct reference timings but
does not enforce wall-clock thresholds:

`.nbc` / `.nbi` cache files near `__pycache__`; cache path or permission issues
should be fixed directly rather than by changing numerical assertions. The
general unittest baseline is 254 tests OK with no skips. Equivalence status
remains 171 pass, 0 xfail, and 0 optional skips.

## Full Module-Family Ledger

Standalone inventory date: 2026-07-05. This table summarizes the current
package families and documented exclusions. It is intentionally conservative:
compiled backend internals, exact C++ memory layout, exact compiled RNG
bitstreams, and source-tree test/build packages are documented exclusions
rather than production requirements for this standalone library.
